发布流程

  1. 点击上方对应页面的“编辑”按钮，确认浏览器已经登录有权限的 GitHub 账号。
  2. 修改文字或结构后，点击页面底部的 Commit changes。
  3. 提交说明建议写清楚，例如“更新作品集案例”或“优化打印简历”。
  4. 回到公开网站刷新查看，GitHub Pages 通常需要 1-3 分钟同步。
建议不要在同一时间让多人编辑同一个页面，避免后提交的人覆盖前一个人的内容。

权限与安全

  1. 协作者需要先登录自己的 GitHub 账号，并接受仓库邀请。
  2. 仓库权限建议给 Write，这样可以编辑和发布，但不会拥有仓库所有权。
  3. 不要把 GitHub token、密码、验证码写进网页、聊天记录或代码文件。
  4. 如果 token 已经发出，请在 GitHub 设置里撤销旧 token，再重新生成。
当前后台不需要填写 token。保存内容依赖 GitHub 登录和仓库协作者权限。

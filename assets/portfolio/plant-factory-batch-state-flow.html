BATCH STATE FLOW SAMPLE

作物批次状态流样例

批次状态流用于回答三个问题：当前批次在哪里、下一步由谁操作、哪些异常会导致暂停或报损。它是生产追溯、工单派发和看板统计的基础。

主状态流

待播种

进入条件

生产计划确认，品种、数量、计划日期和责任人已建立。

系统记录

批次号、目标数量、计划周期、物料准备状态。

育苗中

进入条件

播种完成并完成批次码绑定。

系统记录

苗龄、巡检、补光策略、损耗、异常。

可移栽

进入条件

苗龄、长势或人工复核达到移栽标准。

系统记录

待派工单、目标栽培位、预计转入数量。

定植中

进入条件

移栽工单开始执行，资源位被占用。

系统记录

操作人、栽培位、实际数量、空位和异常。

生长期

进入条件

定植完成并进入生长环境策略。

系统记录

环境参数、设备联动、巡检、异常派单。

待采收

进入条件

达到采收周期或人工判定可采收。

系统记录

计划采收日期、预估产量、采收工单。

已采收/已出库

进入条件

采收、包装、质检、出库动作完成。

系统记录

采收量、商品率、损耗率、追溯码、客户反馈。

异常分支

暂停

  • 设备离线或环境持续越界
  • 工单无法继续执行
  • 需主管复核后恢复

报损

  • 批次质量不达标
  • 病害或环境事故影响
  • 记录原因、数量、责任环节

补录/更正

  • 人工漏记或设备数据缺失
  • 保留修改人和修改原因
  • 影响指标需重新计算
